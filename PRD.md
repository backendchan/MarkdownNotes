# Markdown 笔记应用产品需求文档 (PRD)

## 1. 产品概述和目标用户

### 1.1 产品概述
本产品是一款轻量级、离线优先的 Markdown 笔记应用，旨在为用户提供简洁高效的 Markdown 创作与管理体验。应用采用左右分栏布局，左侧集成笔记管理与编辑功能，右侧提供实时预览，所有数据存储于浏览器本地，确保用户数据隐私与离线可用性。

### 1.2 目标用户
- **主要用户**：需要高效记录与整理信息的知识工作者、学生、技术文档撰写者
- **次要用户**：需要快速编辑 Markdown 内容的开发者、博客作者
- **用户特征**：熟悉 Markdown 语法，注重写作效率，需要离线工作能力，对界面简洁性有较高要求

## 2. 详细的功能列表

### 2.1 笔记管理功能
- 笔记创建：支持新建空白笔记，自动生成唯一标识符与创建时间戳
- 笔记保存：实时自动保存与手动保存双重机制，防止内容丢失
- 笔记删除：支持单个笔记删除，删除前需二次确认
- 笔记列表：按创建时间倒序显示所有笔记，包含标题、修改时间等信息
- 笔记搜索：支持按标题和内容关键词搜索，实时显示匹配结果
- 笔记排序：支持按创建时间、修改时间、标题字母顺序排序

### 2.2 编辑器功能
- Markdown 语法支持：完整支持标准 Markdown 语法（标题、列表、链接、图片、引用等）
- 实时预览：右侧窗格实时渲染 Markdown 内容，所见即所得
- 代码块语法高亮：支持多种编程语言的代码块语法高亮显示
- 编辑器工具栏：提供常用 Markdown 格式快捷按钮（粗体、斜体、列表等）
- 标题与内容分离：独立的标题输入区域与内容编辑区域
- 自动保存：编辑器内容变化时自动保存，避免意外丢失

### 2.3 数据管理功能
- LocalStorage 存储：所有笔记数据安全存储于浏览器 LocalStorage
- 数据导出：支持将单个或全部笔记导出为 Markdown 文件
- 数据导入：支持导入 Markdown 文件创建新笔记
- 数据备份：提供手动备份与恢复功能

### 2.4 用户界面功能
- 布局切换：支持左右分栏（编辑/预览）与全屏编辑模式切换
- 主题切换：支持明/暗两种主题模式
- 响应式设计：适配不同屏幕尺寸，在移动设备上提供优化体验
- 编辑区域调整：支持拖动调整左右分栏宽度比例

## 3. 功能优先级（MVP 和后续版本）

### 3.1 MVP（最小可行产品）功能
- 笔记创建、保存、删除功能
- 基础 Markdown 语法支持
- 左右分栏布局（左侧编辑器+笔记列表，右侧预览）
- LocalStorage 数据存储
- 标题与内容编辑
- 基础代码块显示（无语法高亮）
- 笔记列表显示与简单搜索

### 3.2 后续版本功能
- **V1.1**
  - 代码块语法高亮
  - 编辑器工具栏
  - 笔记排序功能

- **V1.2**
  - 数据导出/导入功能
  - 高级搜索（按标签、时间范围）
  - 编辑区域宽度调整
  - 快捷键支持

- **V1.3**
  - 标签管理功能
  - 笔记分类/文件夹功能
  - 数据云同步选项
  - 版本历史记录

## 4. 界面设计要求

### 4.1 整体布局
- 采用三区域布局：左侧笔记列表区（宽度约25%）、中间编辑区（宽度约35%）、右侧预览区（宽度约40%）
- 支持通过拖拽分隔线调整各区域宽度
- 移动设备上默认采用单区域切换模式，可通过顶部标签切换列表/编辑/预览

### 4.2 色彩方案
- 主色调：采用蓝色系（#3498db）作为主色调，传达专业、可靠的产品形象
- 辅助色：使用灰色系作为背景和文本颜色，确保内容可读性
- 支持明/暗两种主题模式切换
- 语法高亮配色方案需确保代码可读性，区分不同语法元素

### 4.3 组件设计
- **笔记列表**：卡片式设计，显示标题、修改时间、内容预览（前30字符）
- **编辑器**：简洁无干扰的编辑区域，支持行号显示
- **预览区**：清晰展示渲染后的 Markdown 内容，保持与编辑区同步滚动
- **工具栏**：悬浮式或固定顶部的工具栏，包含常用格式按钮
- **搜索框**：位于笔记列表顶部，支持实时搜索过滤

### 4.4 交互设计
- 笔记创建：右上角明显的"+"按钮
- 笔记删除：每条笔记hover时显示删除按钮，点击后二次确认
- 编辑切换：点击列表中的笔记自动在编辑区加载内容
- 保存状态：显示实时保存状态指示器
- 空状态：无笔记时显示引导创建笔记的友好提示

## 5. 技术栈建议

### 5.1 前端框架
- 主框架：React.js 或 Vue.js，提供组件化开发和高效DOM更新
- 状态管理：Redux（React）或 Vuex（Vue）管理应用状态
- 路由管理：React Router 或 Vue Router 处理视图切换

### 5.2 Markdown处理
- Markdown解析：marked.js 或 markdown-it 用于将Markdown转换为HTML
- 语法高亮：Prism.js 或 highlight.js 实现代码块语法高亮
- 编辑器组件：可考虑使用已有的Markdown编辑器组件如 react-markdown-editor-lite 或 vue2-editor

### 5.3 本地存储
- LocalStorage API 用于数据持久化存储
- 可考虑使用 localForage 库优化LocalStorage使用体验

### 5.4 UI组件库
- 基础组件：Ant Design、Element UI 或 Material-UI
- 图标库：Font Awesome 或 Ant Design Icons

### 5.5 构建工具
- 打包工具：Webpack 或 Vite
- 代码规范：ESLint + Prettier
- 测试工具：Jest + React Testing Library/Vue Test Utils

## 6. 非功能性需求

### 6.1 性能要求
- 页面加载时间：首次加载<2秒，后续加载<1秒
- 编辑器响应：输入延迟<100ms
- 预览更新：编辑内容后预览区更新延迟<300ms
- 搜索响应：搜索操作响应时间<500ms，支持即时搜索

### 6.2 兼容性要求
- 浏览器支持：Chrome（最新版）、Firefox（最新版）、Safari（最新版）、Edge（最新版）
- 响应式支持：桌面端（≥1200px）、平板（768px-1199px）、移动端（<768px）

### 6.3 安全要求
- 数据安全：所有数据存储在用户本地，不上传至服务器
- XSS防护：对用户输入内容进行安全处理，防止跨站脚本攻击
- 数据隔离：不同用户（浏览器配置文件）数据完全隔离

### 6.4 可用性要求
- 离线可用：完全支持离线工作模式
- 操作容错：提供撤销/重做功能，删除操作需二次确认
- 可访问性：支持键盘导航，符合WCAG 2.1 AA级标准

### 6.5 可维护性要求
- 代码结构：模块化设计，清晰的代码注释
- 错误处理：完善的错误捕获和友好的错误提示
- 日志记录：关键操作记录本地日志，便于问题排查
